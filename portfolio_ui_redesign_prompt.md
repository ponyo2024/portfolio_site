# Portfolio 网站 UI 重构 Prompt

## 重构方向

当前网站的核心问题是：**像一个"关于我"的个人介绍页，而不是一个"作品展示"的portfolio站。** 需要彻底调整信息架构和视觉重心。

## 信息架构调整

### 删除以下内容：
- ❌ 删除整个 About section（个人介绍、技能矩阵、教育背景）——这些信息在简历里已经有了，网站不需要重复
- ❌ 删除 Experience section（工作经历时间线）——同上
- ❌ Hero 区域的名字不需要巨大展示，弱化个人信息，强化项目

### 保留并强化：
- ✅ 项目展示区域——这是整个网站的核心，应该占 90% 的篇幅
- ✅ Contact section——保留但精简
- ✅ 项目详情子页

### 调整后的首页结构（从上到下）：

```
1. Mini Header / Nav Bar（固定顶部）
   - 左侧：名字（小字）+ 一句话定位
   - 右侧：导航锚点（项目 / 联系）+ 下载PDF按钮

2. Hero（极简，不超过半屏高度）
   - 一句有力的标语（不是自我介绍，而是价值主张）
   - 例如："从真实业务问题出发，构建AI产品原型"
   - 不要放名字大字、不要放个人照片、不要放技能标签

3. Projects（核心区域，占绝大部分页面）
   - 这里要彻底重做，见下方详细设计

4. Contact（精简）
   - 一行联系方式即可，不需要大标题和长文案
```

## 项目展示区域——彻底重做

### 当前问题：
- 项目卡片全是文字，没有视觉内容（截图/录屏）
- 卡片太小太挤，信息层次不清
- 分类标签（Agent × 工具调用 等）占了太多视觉权重，喧宾夺主

### 重做方案：

#### 首页项目卡片——大图优先

每个项目卡片应该是 **视觉驱动** 的，不是文字驱动的。参考 Dribbble / Behance 的作品展示风格：

```
┌─────────────────────────────────────────┐
│                                         │
│          项目截图 / 录屏封面              │
│          （占卡片 60-70% 面积）            │
│                                         │
├─────────────────────────────────────────┤
│  项目名称                                │
│  一句话描述                    →查看详情   │
│  [Agent] [Tool Use] [FastAPI]           │
└─────────────────────────────────────────┘
```

**关键设计要求：**

1. **每张卡片必须有一个视觉主体**——截图、产品界面、架构图、或录屏封面。如果还没有真实截图，先用 placeholder 图片（纯色背景 + 项目名），但卡片结构要预留图片位置且图片区域要大
2. **Featured 项目用大卡片**（宽度占满或占 2/3），非 Featured 项目用标准卡片
3. **卡片布局用 Grid**，建议：
   - Featured: 一行一个，宽卡片，图片在左文字在右（或图片在上文字在下）
   - 标准: 一行两个或三个
4. **分类标签弱化**——不要用大标题 + 分隔线的方式分组。改为在卡片上用小标签区分即可，或者用 Tab/Filter 切换
5. **Hover 效果**——卡片 hover 时图片区域轻微放大或加遮罩，暗示可点击

#### 项目卡片中的图片/媒体处理

在 meta.json 中已有 thumbnail 字段。请确保：

- 图片区域有固定宽高比（建议 16:9 或 3:2）
- 使用 `object-cover` 确保图片不变形
- 如果 thumbnail 不存在，显示一个美观的 placeholder：项目名 + accent color 背景 + 简单的图标或图案
- 后续我会替换成真实截图

#### 新增：支持视频/GIF 展示

在 meta.json 中新增一个可选字段：

```json
{
  "media": {
    "type": "image" | "video" | "gif",
    "src": "./assets/demo.mp4",
    "poster": "./assets/thumbnail.png"
  }
}
```

在项目详情页中，如果有 video 类型的 media，在页面顶部显示可播放的 demo 视频（替代静态截图）。首页卡片只展示 poster/thumbnail 静态图。

## 项目详情页优化

### 当前问题：
- 纯文字堆砌，像一篇 blog post
- 没有视觉元素打断文字流
- 标题层次不够清晰

### 优化方向：

```
┌──────────────────────────────────────────────┐
│  ← 返回                                      │
│                                              │
│  [Agent × 工具调用]           已上线           │
│  Game Market Intelligence Agent               │
│  用AI Agent自动生成游戏竞品周报                  │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │                                      │    │
│  │     Demo 视频 / 主截图（大幅展示）      │    │
│  │                                      │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  [FastAPI] [React] [DashScope] [SensorTower] │
│  [Live Demo →]  [GitHub →]                   │
│                                              │
│  ─────────────────────────────────────────   │
│                                              │
│  Case Study 正文（MDX 渲染）                   │
│  - 正文中也应该穿插截图和架构图                   │
│  - 用分隔线或卡片区分不同章节                     │
│                                              │
│  ─────────────────────────────────────────   │
│                                              │
│  版本更新日记（可展开/收起）                      │
│                                              │
│  ─────────────────────────────────────────   │
│                                              │
│  ← 上一个项目          下一个项目 →             │
│                                              │
└──────────────────────────────────────────────┘
```

**关键改进：**

1. **顶部放一张大的 hero 图/视频**——让人第一眼看到产品长什么样
2. **Live Demo 和 GitHub 链接要醒目**——用按钮而不是文字链接
3. **正文排版优化**：
   - 每个 section 标题用粉色左边框或粉色下划线区分
   - 段落间距加大，不要太密
   - MDX 中的图片自动居中、加圆角和微阴影
4. **Tech stack 标签放在 hero 下方**，不要放在正文里

## 视觉风格优化

### 整体节奏感
- 现在的问题是"所有元素大小差不多，找不到重点"
- 要建立 **大-小-大** 的视觉节奏：大图项目卡 → 小字标签 → 大标题 → 正文
- Section 之间用足够的留白分隔（至少 120px），不要用分隔线

### 粉色使用
- 当前粉色使用太多太均匀——标题、副标题、标签、按钮全是粉色
- 粉色应该只用在 **最重要的交互节点**：CTA 按钮、hover 状态、当前选中状态
- 标题用白色，副标题用灰色，只有关键词或链接用粉色点缀
- 粉色的 accent 效果是"万绿丛中一点红"——用得越少，越醒目

### 字体层次
- 项目名：大号、粗体、白色
- 副标题/描述：中号、常规、浅灰
- 标签/技术栈：小号、mono 字体、深灰边框
- 正文：舒适的行高（1.8）和段间距

### 卡片设计
- 卡片背景用比页面背景略浅一度的色（如 `#14141F`）
- 不要用明显的边框，用微妙的 `border: 1px solid rgba(255,255,255,0.06)` 
- hover 时边框变粉色（`rgba(255,107,138,0.3)`）
- 卡片圆角统一 12px

## DO NOT
- ❌ 不要保留 About section 和 Experience section
- ❌ 不要在 Hero 区域放大字名字
- ❌ 不要做纯文字的项目卡片——每张卡片必须有图片/视觉区域
- ❌ 不要到处都用粉色——克制使用，只在关键交互点
- ❌ 不要用分隔线分割 section，用留白
- ❌ 不要改变现有的数据结构和内容读取逻辑，只改 UI 层

## 执行优先级

1. 首先：删除 About 和 Experience section
2. 其次：重做 Hero 区域（极简化）
3. 然后：重做项目卡片（大图优先布局）
4. 接着：优化项目详情页（顶部大图 + 排版改进）
5. 最后：全局视觉细节（粉色使用、间距、hover 效果）
